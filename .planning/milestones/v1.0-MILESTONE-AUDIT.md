## Tech Debt Summary

No critical blockers. 6 deferred items across 4 phases.

### Phase 3: Normalization Layer

- **GUID_VALUE_KEYS empty frozenset** — GUID keys in Alteryx `.yxmd` configuration have not been confirmed from real files. The stripping mechanism (`_strip.py`) is implemented; the key registry (`patterns.py: GUID_VALUE_KEYS`) is empty. GUIDs embedded in tool config fields are not currently neutralized. If real workflows contain session GUIDs in tool configs, `config_hash` comparisons may produce false-positive modification diffs. Tracked: `tests/test_normalizer.py::test_guid_key_stripped_to_sentinel` xfail. **Fix:** Inspect real `.yxmd` files and populate `GUID_VALUE_KEYS` with confirmed GUID field names.

### Phase 6: Pipeline Orchestration and JSON Renderer

- **JSONRenderer schema divergence** — `JSONRenderer.render()` produces `{summary: {...}, tools: [...], connections: [...]}`. The CLI `--json` path uses `_cli_json_output()` which produces `{added: [...], removed: [...], modified: [...], metadata: {...}}`. These schemas are incompatible. `JSONRenderer` is exercised only by its 5 unit tests, not by any CLI invocation. If downstream tooling is built against `JSONRenderer`, it will be incompatible with actual `acd --json` output. **Fix:** Either wire `JSONRenderer` into the CLI `--json` path, or document the CLI JSON schema as the stable contract and deprecate `JSONRenderer`.

### Phase 8: Visual Graph

- **Browser-interactive behaviors** require human verification: click-to-diff slide panel, hover tooltip rendering, show-only-changes toggle, fit-to-screen animation. Code is structurally correct; cannot be verified without browser runtime.

### Phase 9: CLI Entry Point

- **Spinner TTY behavior** — animated spinner in real terminal requires manual test (CliRunner does not emulate TTY).
- **--canvas-layout visual layout** — requires browser rendering to confirm Alteryx X/Y coordinate positioning vs hierarchical layout. Automated test confirms `canvas_positions()` returns correct coordinates.
- **Governance footer UX** — `<details>` disclosure element expand/collapse and readability of SHA-256 values requires browser test.

### Total: 6 items across 4 phases
