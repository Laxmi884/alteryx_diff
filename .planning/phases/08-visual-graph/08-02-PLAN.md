**Class signature:**
```python
class GraphRenderer:
    """Render a DiffResult as an interactive vis-network graph HTML fragment.

    Produces: <div id="graph-wrapper"> + inline <style> + inline <script>
    Fragment is embedded by HTMLRenderer via {{ graph_html | safe }}.
    vis-network standalone UMD is inlined — zero CDN references.
    Physics is disabled; positions are pre-computed in Python.
    """

    def render(
        self,
        result: DiffResult,
        all_connections: tuple[AlteryxConnection, ...],
        nodes_old: tuple[AlteryxNode, ...],
        nodes_new: tuple[AlteryxNode, ...],
        *,
        canvas_layout: bool = False,
    ) -> str:
```

**render() logic:**
1. Compute `all_nodes = nodes_old + nodes_new` — pass to `build_digraph` (it deduplicates by using the last-seen node per tool_id in the node set; NOTE: all_nodes may have duplicates for unchanged nodes; build_digraph iterates all_nodes so it processes some IDs twice — this is OK since status assignment is idempotent. If you want clean dedup: build a dict keyed by int(n.tool_id) taking the new node over old.)
   Actually: deduplicate as `dict[int, AlteryxNode]` — old nodes first, new nodes override:
   ```python
   nodes_map: dict[int, AlteryxNode] = {}
   for n in nodes_old:
       nodes_map[int(n.tool_id)] = n
   for n in nodes_new:
       nodes_map[int(n.tool_id)] = n
   all_nodes_tuple = tuple(nodes_map.values())
   ```
2. `G = build_digraph(result, all_connections, all_nodes_tuple)`
3. Compute positions:
   - If `canvas_layout`: `positions = canvas_positions(nodes_old, nodes_new)`
   - Else: `positions = hierarchical_positions(G)`
4. Build `nodes_json` list:
   ```python
   nodes_json = []
   for node_id, (px, py) in positions.items():
       attrs = G.nodes[node_id]
       entry: dict[str, Any] = {
           "id": node_id,
           "label": attrs["label"],
           "color": attrs["color"],
           "status": attrs["status"],
           "title": attrs["title"],  # vis-network tooltip
           "x": px,
           "y": py,
           "fixed": True,
       }
       nodes_json.append(entry)
   ```
5. Build `edges_json` list:
   ```python
   edges_json = [
       {"id": f"{src}-{dst}", "from": src, "to": dst}
       for src, dst in G.edges()
   ]
   ```
6. Build `field_counts`: `dict[int, int]` mapping tool_id -> number of changed fields (for tooltip). For modified nodes: `{int(nd.tool_id): len(nd.field_diffs) for nd in result.modified_nodes}`. For added/removed: use 0 (or omit).
7. Update tooltip for modified nodes: `title = f"{label} | modified | {count} field(s) changed"`. Do this by iterating nodes_json after construction and enriching the `title` field for `status == "modified"`.
8. Load vis-network JS: `vis_js = load_vis_js()`
9. Render via Jinja2:
   ```python
   env = Environment(autoescape=True)
   env.policies["json.dumps_kwargs"] = {"ensure_ascii": False}
   template = env.from_string(_GRAPH_FRAGMENT_TEMPLATE)
   return template.render(
       nodes_json=json.dumps(nodes_json),
       edges_json=json.dumps(edges_json),
       vis_js=vis_js,
   )
   ```
   NOTE: `nodes_json` and `edges_json` are passed as already-serialized strings (not dicts) so they can be embedded as raw JS without Jinja2 escaping. Use `{{ nodes_json | safe }}` in the template.

- The vis-network JS (`{{ vis_js | safe }}`) must be injected INSIDE the IIFE function scope to avoid polluting window globals
- `nodes_json` and `edges_json` are pre-serialized Python strings passed with `| safe` — do NOT double-encode
- ALL DOM manipulation uses `createElement`/`textContent` — NO innerHTML (matches Phase 7 pattern)
- The fragment must NOT contain `<html>`, `<head>`, or `<body>` tags — it is a section, not a document
- `autoescape=True` on Jinja2 Environment (ruff B701) — node/edge data passed as pre-serialized `| safe` strings, not as dicts through tojson filter
- ruff noqa: E501 for the file header — template lines exceed 88 chars
- mypy: `dict[str, Any]` for all intermediate dicts; `list[dict[str, Any]]` for nodes_json/edges_json; use `json.dumps` (not tojson) for the pre-serialized JSON strings
