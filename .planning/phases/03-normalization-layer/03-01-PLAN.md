- `source: AlteryxNode` — carry the original node for downstream stages that need identity fields (tool_id, tool_type)
- `config_hash: ConfigHash` — SHA-256 hex digest, produced by the normalizer (not computed here)
- `position: tuple[float, float]` — (x, y) carried from source.x, source.y; this is the SEPARATE data path from config_hash

NormalizedWorkflowDoc:
- `source: WorkflowDoc` — carry the original for downstream metadata access
- `nodes: tuple[NormalizedNode, ...]` — all nodes replaced by their normalized forms
- `connections: tuple[AlteryxConnection, ...]` — preserved unchanged from source WorkflowDoc

Module-level docstring: "Normalized pipeline output models produced by the normalization stage."

Do NOT import from `alteryx_diff.models` (would cause circular import — normalized.py is INSIDE models/).
Do NOT add `__all__` in this file — the __init__.py is the public surface.
Do NOT add any methods, properties, or logic — pure frozen dataclasses only.
