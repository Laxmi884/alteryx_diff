### CRITICAL: POST /api/save/undo — Never Called from Frontend

**Affected requirement:** SAVE-02 (undo last saved version)

`app/routers/save.py` line 49 implements `POST /api/save/undo` (calls `git_undo_last_commit`, returns `has_any_commits`). The route is registered, the backend is tested GREEN (test_save.py). **However**, `HistoryPanel.tsx` line 705 Undo button fires `onUndo` → `AppShell.handleUndo()` (lines 161-164), which only refreshes state:

```tsx
async function handleUndo() {
  await fetchWatchStatus()   // ← only fetches current status
  await fetchHistory()        // ← only fetches commit list
  // POST /api/save/undo is NEVER called
}
```

The `git_undo_last_commit` backend call never executes. The button appears functional (no error) but the last git commit is never removed. This is a critical integration gap — SAVE-02 is **unsatisfied** at the E2E level despite passing backend unit tests.

**Fix:** Add `await fetch('/api/save/undo', { method: 'POST', ... })` inside `AppShell.handleUndo()` before the state refresh calls, passing `activeProject.id` and `activeProject.folder` in the body.

### LOW: mergeBaseSha Prop Silently Dropped at HistoryPanel

**Affected requirement:** BRANCH-03 (partial impact)

`AppShell` passes `mergeBaseSha` prop to `HistoryPanel` at lines 223 and 255, but `HistoryPanelProps` TypeScript interface does not declare this prop — it is silently ignored. The "vs main" compare toggle works because `DiffViewer` is rendered directly via the `hasCommits && selectedDiff` branch in AppShell (not from within `HistoryPanel`). No runtime failure; functionally correct for the common case.

**Fix:** Declare `mergeBaseSha?: string` in `HistoryPanelProps` and use it where the compare toggle is wired.
